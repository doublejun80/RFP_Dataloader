# PRD: Tauri RFP v2

## 제품 한 줄 정의

로컬 RFP 문서를 OpenDataLoader와 선택형 LLM으로 구조화하고, 구매/제안 담당자가 필요한 자원, 수량, 인력, 업무 범위, 납품물, 검수 조건, 리스크를 원문 근거와 함께 검토하는 데스크톱 앱.

## 사용자

| 사용자 | 해야 할 일 | v2가 제공할 것 |
|---|---|---|
| 구매 담당자 | 무엇을 사야 하는지 파악 | 구매 항목, 수량, 스펙, 예산 영향 |
| 제안 담당자 | 제안 범위와 리스크 판단 | 업무 범위, 제출물, 평가/계약 조건 |
| PM/기술 리더 | 인력과 일정 산정 | 역할/MM/상주/일정/검수 조건 |
| 검토자 | 자동 추출값 확인 및 수정 | 근거 기반 검토 화면과 correction 기록 |

## MVP 목표

1. 사용자가 로컬 RFP PDF를 추가한다.
2. 앱이 OpenDataLoader로 Markdown/JSON을 생성한다.
3. 앱이 핵심 후보 블록을 만들고 SQLite에 저장한다.
4. 사용자가 LLM 사용을 켠 경우, 후보 블록을 OpenAI 또는 Gemini에 보내 구조화 JSON을 받는다.
5. Rust 검증기가 결과를 검사하고 blocker/warning을 만든다.
6. UI는 "분석 생성됨", "검토 필요", "확정 가능"을 분리해 보여준다.
7. 결과는 Markdown/Docx/JSON으로 내보낼 수 있다.

## 비목표

- PySide6 유지.
- v1 정규식 추출기를 그대로 이식.
- 원격 문서 업로드형 분석.
- 자체 사용자 계정/로그인.
- 여러 RFP의 영업 매칭/추천 서비스.
- 나라장터 자동 수집. 이 기능은 후속 collector로 분리한다.
- 검색 가능한 PDF 생성.

## 기능 요구사항

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| FR-001 | 로컬 문서 등록 | PDF 파일을 프로젝트로 등록하고 원본 경로, 해시, 크기를 DB에 저장 |
| FR-002 | OpenDataLoader 추출 | JSON/Markdown output과 run log를 저장 |
| FR-003 | 원문 블록 저장 | page, element type, text, bbox를 `document_blocks`에 저장 |
| FR-004 | RFP 프로젝트 생성 | 한 문서에 대해 `rfp_projects`와 핵심 필드 후보를 생성 |
| FR-005 | LLM opt-in 분석 | 사용자가 provider/API key/model을 설정한 경우에만 LLM 실행 |
| FR-006 | 구조화 추출 | 사업정보, 요구사항, 구매항목, 인력, 납품물, 검수, 리스크를 저장 |
| FR-007 | 근거 연결 | 모든 추출 entity는 최소 1개 이상의 evidence link를 가져야 함 |
| FR-008 | 검증 게이트 | blocker가 있으면 완료가 아니라 `검토 필요`로 표시 |
| FR-009 | 수동 보정 | 사용자가 필드/요구사항/항목 값을 수정하면 원값과 새 값을 기록 |
| FR-010 | 내보내기 | Markdown, JSON, Docx export를 생성하고 export 이력을 저장 |

## 비기능 요구사항

| 항목 | 기준 |
|---|---|
| 로컬 우선 | 문서 원본은 외부로 전송하지 않음 |
| LLM 사용 | 명시적 opt-in, 후보 텍스트만 전송, 원본 파일 전송 금지 |
| 추적성 | 모든 핵심 값은 원문 evidence로 돌아갈 수 있어야 함 |
| 재현성 | 같은 입력과 같은 설정은 같은 candidate/validation 결과를 생성 |
| 성능 | 100쪽 PDF 1건은 추출/저장/후보 생성까지 사용자가 기다릴 수 있는 시간 안에 완료 |
| 안정성 | 실패해도 원본 파일과 이전 분석 결과는 보존 |
| 보안 | API key는 DB plain text 저장 금지 |

## 성공 기준

- 실제 RFP bundle 기준 각 문서가 생성 성공 여부와 품질 blocker를 분리해 보고한다.
- "요구사항 0건"이 나오면 완료가 아니라 blocker다.
- 구매 항목/인력/납품물/검수/리스크 중 하나라도 근거 없는 값은 확정값으로 표시하지 않는다.
- 사용자는 특정 행을 클릭해 원문 page/block/bbox 근거를 확인할 수 있다.
